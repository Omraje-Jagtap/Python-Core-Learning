This repository documents my structured progression through Python programming. It serves as my technical foundation for AI/ML Engineering, tracking my growth from basic syntax to object-oriented design, file handling, and applied problem-solving.

Coming from a C programming background, this repo also marks a deliberate transition — trading manual memory and pointer-level thinking for Python's abstraction, while keeping the underlying logic discipline intact. That combination is the actual advantage: I don't just know *that* something works, I know *why* it works underneath.

---

## 📁 Repository Architecture

My work is organized into logical sub-folders to maintain clean engineering architecture:

| Folder | Purpose |
|---|---|
| **`/basic-program`** | Foundational logic, syntax, and control flow practice using loops and conditions. |
| **`/recursion`** | Exploring recursive function logic, understanding base cases, and breaking down iterative patterns. |
| **`/function`** | Modular code blocks tracking functional logic, argument passing, and clean value execution arrays. |
| **`/OOP`** | Practical code implementing the pillars of Object-Oriented Programming (Encapsulation, Inheritance, Polymorphism, and Abstraction) to understand how advanced ML frameworks function under the hood. |
| **`/file-IO`** | Pipeline scripts parsing external data files (like `.txt` or `.csv` arrays) and implementing crash-safe `try...except` exception handling. |
| **`/hackerrank`** | Applied problem-solving — HackerRank challenge solutions organized by domain (Python fundamentals, algorithms, data structures), used to stress-test concepts learned in the folders above under timed, competitive conditions. |

Each folder is self-contained and grouped by *topic*, not by date — the goal is a repo someone can navigate to learn a concept, not a diary of when I happened to write the code.
